Projeto Prático: Sistema para Locadora de Livros

Contextualização
O acesso à leitura é essencial para o desenvolvimento cultural e educacional. Uma locadora de livros permite que leitores tenham acesso temporário a diversas obras sem a necessidade de comprá-las. Este projeto propõe a construção de um sistema simples para gerenciamento de empréstimos de livros, utilizando a linguagem Java e conceitos de Programação Orientada a Objetos (POO), com foco em persistência de dados e controle de operações. O sistema será utilizado via console, sem interface gráfica.

Objetivos
    • Simular o funcionamento básico de uma locadora de livros.
    • Aplicar os conceitos de Programação Orientada a Objetos (POO).
    • Implementar o padrão de arquitetura MVC.
    • Trabalhar com coleções (List, Map, Set) e persistência de dados.
    • Garantir controle de empréstimos e devoluções.

Funcionalidades do Sistema
Módulo de Livros:
    • Cadastro de livros com título, autor, ID único e status de disponibilidade.
    • Listagem de todos os livros e filtragem por disponibilidade.
Módulo de Clientes:
    • Cadastro de clientes com nome e ID único.
    • Verificação de histórico de empréstimos.
Módulo de Empréstimo:
    • Empréstimo de livros disponíveis a clientes cadastrados.
    • Registro de data do empréstimo e previsão de devolução.
    • Devolução de livros com atualização do status.
Persistência:
    • Armazenamento de dados de livros, clientes e empréstimos em arquivos.
    • Geração de relatório de livros emprestados em arquivo .txt.

Estrutura Técnica
Tecnologias Utilizadas:
    • Linguagem: Java (versão 8 ou superior)
    • Entrada/Saída: Console (sem interface gráfica)
    • Arquivos:
        ◦ .txt para relatórios e controle de devoluções
        ◦ .dat para persistência binária de dados
Arquitetura:
    • Modelo (Model):
        ◦ Pessoa (abstrata)
        ◦ Cliente
        ◦ Livro
        ◦ Emprestavel (interface)
    • Controle (Controller):
        ◦ LocadoraController
    • Visão (View):
        ◦ Classe Main com menu interativo via console

Requisitos Funcionais
RF01 – O sistema deve permitir o cadastro de livros.
RF02 – O sistema deve permitir o cadastro de clientes.
RF03 – O sistema deve registrar e controlar empréstimos e devoluções.
RF04 – O sistema deve verificar se o livro está disponível antes do empréstimo.
RF05 – O sistema deve gerar relatórios de livros emprestados.
RF06 – O sistema deve persistir os dados em arquivos .txt e .dat.
RF07 – O sistema deve utilizar herança, interface e coleções.

Requisitos Não Funcionais
    • O sistema não deve possuir interface gráfica.
    • A arquitetura deve seguir o padrão MVC.
    • O código deve estar organizado em classes separadas.
    • O projeto deve ser publicado em um repositório público no GitHub.

Solução Proposta
Projeto: Sistema para Locadora de Livros 
    • Cadastro e controle de livros e clientes.
    • Registro de empréstimos e devoluções com verificação de disponibilidade.
    • Utilização de coleções (Map, Set, List) para gerenciamento de dados.
    • Persistência em arquivos para garantir integridade dos dados entre sessões.
    • Interface de linha de comando amigável e intuitiva.
    • Separação clara entre modelo, controle e visualização.
